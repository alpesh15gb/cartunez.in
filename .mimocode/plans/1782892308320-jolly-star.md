# Product Detail Page (PDP) — Like HotTracks.in

## Goal
Add a full product detail page at `/product/:handle` with image gallery, variant selectors, add to cart, description, and related products — matching the hottracks.in layout.

## Current State
- Single-page app (no routing, no react-router-dom)
- Products open via quick view modal only
- `fetchProduct(handle)` in `lib/medusa.ts` already fetches a single product from Medusa
- Product interface in `useProducts.ts` doesn't include `images` field (Medusa returns it)
- Nginx already has `try_files $uri $uri/ /index.html` SPA fallback

## Implementation Steps

### Step 1: Install react-router-dom

```bash
cd frontend && npm install react-router-dom
```

### Step 2: Extend Product interface with images

**File:** `frontend/src/hooks/useProducts.ts`

Add `images` to the Product interface:
```ts
images: Array<{ id: string; url: string; }>;
```

### Step 3: Create ProductDetailPage component

**File:** `frontend/src/pages/ProductDetailPage.tsx` (new)

Layout (matching hottracks.in):
```
┌─────────────────────────────────────────────────────┐
│ Breadcrumb: Home > Category > Product Title          │
├──────────────────────┬──────────────────────────────┤
│                      │  Product Title                │
│  Main Image          │  Price                       │
│  (large, clickable   │  ─────────────────           │
│   to zoom)           │  Option: [Size] [Color] ...  │
│                      │  Qty: [- 1 +]                │
│  Thumbnails ──────►  │  [Add to Cart] [Wishlist]    │
│  [img] [img] [img]   │                              │
├──────────────────────┴──────────────────────────────┤
│ Description / Details / Vehicle Compatibility tabs   │
├─────────────────────────────────────────────────────┤
│ Related Products (same category)                     │
└─────────────────────────────────────────────────────┘
```

Key features:
- **Image gallery**: Main image + thumbnail strip below. Click thumbnail to swap main image. Click main image to view fullscreen.
- **Variant selectors**: Per-option buttons (like existing quick view but full-size). Show price update when variant changes.
- **Quantity selector**: +/- buttons with number input
- **Add to Cart**: Full-width button, calls `addItem(variantId)`
- **Vehicle compatibility badge**: Shows compatible vehicles if data exists
- **Breadcrumb**: Home > Category Name > Product Title
- **Related products**: Fetch products from same category, show 4 in a grid

### Step 4: Set up routing

**File:** `frontend/src/main.tsx`

Wrap App in BrowserRouter and add routes:
```tsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import ProductDetailPage from './pages/ProductDetailPage';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <BrowserRouter>
      <AuthProvider>
        <CartProvider>
          <Routes>
            <Route path="/product/:handle" element={<ProductDetailPage />} />
            <Route path="*" element={<App />} />
          </Routes>
        </CartProvider>
      </AuthProvider>
    </BrowserRouter>
  </StrictMode>,
)
```

### Step 5: Update ProductCard to link to PDP

**File:** `frontend/src/App.tsx`

In `ProductCard`, wrap the image area and title with a `<Link to={/product/${product.handle}}>` instead of `onClick={onQuickView}`. Keep the "Quick View" hover overlay as a secondary action.

Also update the quick view modal's product title to be a link to the PDP.

### Step 6: Add useProduct hook for PDP

The existing `useProduct` hook in `useProducts.ts` already fetches by handle. It returns the full Medusa product object (with images, options, variants). Just need to extend the interface to include `images`.

### Step 7: Fetch related products

In the PDP, after loading the product, fetch products from the same category:
```ts
const relatedProducts = useProducts({ 
  categoryId: product?.categories?.[0]?.id ? [product.categories[0].id] : undefined, 
  limit: 4 
});
// Filter out current product from results
```
