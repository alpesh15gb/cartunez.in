# Plan: Import NeoWheels Alloy Wheel Catalog

## Overview
Import ~40 wheel designs with 500+ variants from neowheels.com into the Medusa product catalog, including car compatibility data.

## Data Structure from NeoWheels

Each design (e.g., "Hydra") is a product. Each variant within a design is a Medusa product variant:

| Field | Example | Maps To |
|-------|---------|---------|
| Design | Hydra | `product.title`, `product.handle` |
| Size | 18x7.5 | Product option "Size" |
| PCD | 5x139.7 | Product option "PCD" |
| Finish | CB+M | Product option "Finish" |
| MRP | ₹16615 | `variant.metadata.mrp` |
| Sale Price | ₹15615 | `variant.prices[0].amount` |
| Offset | 35 | `variant.metadata.offset` |
| Bore Size | 70.1 | `variant.metadata.bore_size` |
| Image | URL | Downloaded and uploaded |
| Car compatibility | Mahindra XUV 5OO | `product_vehicle_compatibility` rows |

## Implementation Steps

### Step 1: Create the scraper script
**File**: `backend/cartunez-medusa/scripts/scrape-neowheels.js`

A standalone Node.js script that:
1. Fetches `https://www.neowheels.com/product` to get all design slugs (~40 designs)
2. For each design, fetches `https://www.neowheels.com/product/{slug}` to get all variant links
3. For each variant, fetches the detail page to extract: size, PCD, finish, price, MRP, offset, bore size, image URL, and car compatibility list
4. Saves scraped data to a JSON file (`neowheels-data.json`) for review before import

Uses `fetch` (Node 18+) and `cheerio` for HTML parsing. Add `cheerio` as a dev dependency in package.json.

### Step 2: Create the import script
**File**: `backend/cartunez-medusa/scripts/import-neowheels.js`

A standalone Node.js script that:
1. Loads the scraped JSON data
2. Bootstraps the Medusa container (same pattern as `seed-data.js`)
3. Creates/ensures an "Alloy Wheels" category exists
4. For each design:
   - Creates a Medusa product with options: Size, PCD, Finish
   - For each variant: creates a product variant with correct pricing and metadata
   - Downloads the product image and uploads it via Medusa's file service
   - Links the product to compatible vehicles via `product_vehicle_compatibility`
5. Uses find-or-create pattern (idempotent — safe to re-run)

### Step 3: Update the seed data
**File**: `backend/cartunez-medusa/seed-data.js`

Add "Alloy Wheels" to the `categoriesToCreate` array so the category exists before import.

### Step 4: Vehicle compatibility mapping

The neowheels.com car compatibility data uses names like "MAHINDRA XUV 5OO". We need to map these to existing `vehicle_variant` IDs in the database.

Strategy:
1. Query existing vehicle makes/models/variants from the database
2. Fuzzy match neowheels car names to existing vehicle data
3. If no match found, skip that compatibility link (don't create new vehicle records — those are managed by the vehicle-loader)

### Step 5: Image handling

For each product variant:
1. Download the image from neowheels.com
2. Upload via Medusa's local file service (saves to `uploads/` directory)
3. Set as the variant's thumbnail

## Execution Order

1. Run scraper: `docker exec cartunez-medusa node scripts/scrape-neowheels.js`
2. Review `neowheels-data.json` to verify data quality
3. Run importer: `docker exec cartunez-medusa node scripts/import-neowheels.js`
4. Verify products in Medusa admin at `http://localhost:9000/app`

## Verification

1. Check product count in Medusa admin
2. Verify a sample product has correct variants, prices, and images
3. Test the vehicle compatibility filter on the frontend
4. Check that MeiliSearch reindex picks up the new products

## Notes

- Prices are per wheel (not per set) — match NeoWheels pricing directly
- The scraper handles pagination — neowheels shows all variants on each design page
- The script is idempotent — re-running won't create duplicates
- Images are cached locally — re-running won't re-download existing images
